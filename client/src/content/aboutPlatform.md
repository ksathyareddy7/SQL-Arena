### What is SQL Arena?

SQL Arena is a **local-first, real-world SQL practice platform** designed to help you go beyond tutorials and actually **think like a data engineer / analyst**.

Instead of toy datasets, you practice on **large, realistic schemas** across multiple domains:

* Social media
* Ecommerce
* Food delivery
* Ride-hailing
* Movies / streaming
* Banking
* Real estate
* Healthcare
* Education
* Chat systems

Each domain is structured to simulate **real production databases**, so the problems you solve feel like actual work scenarios — not textbook exercises.

---

### Why this project exists

Most SQL learning platforms have a few problems:

* Small, unrealistic datasets
* Repetitive or shallow questions
* No focus on real-world thinking
* Too dependent on SaaS / internet

SQL Arena was built to solve that.

#### The goal is simple:

> **Help you become genuinely strong at SQL — not just pass interviews.**

This platform focuses on:

* Real-world query patterns
* Complex joins and edge cases
* Performance-aware thinking
* Multi-step analytical problems

---

### How to use SQL Arena

#### 1. Pick a domain (App)

Choose a dataset like:

* Social → posts, likes, followers
* Bank → transactions, KYC, accounts
* Hospital → admissions, claims, doctors

Each app has its own schema.

---

#### 2. Explore the schema

Before solving:

* Use the **Schema Explorer**
* Understand relationships
* Identify keys and joins

This step is critical — just like real work.

---

#### 3. Solve challenges

* Read the problem carefully
* Write your query in the editor
* Use **Run** to test
* Use **Submit** to validate

---

#### 4. Use hints (if needed)

Hints are progressive:

1. Direction
2. Logic hint
3. Near-solution hint

Use them wisely — they affect scoring in strict modes.

---

#### 5. Review solutions

After solving (or giving up), explore:

* Multiple approaches
* Tradeoffs
* Better patterns

---

#### 6. Track your progress

* See solved vs attempted
* Monitor success rate
* Build streaks

---

### How SQL Arena works (under the hood)

* Runs on **PostgreSQL**
* Each app has its own schema
* Only **SELECT queries are allowed**
* Results are compared server-side
* Deterministic ordering enforced where required

This ensures:

* Fair evaluation
* Real query correctness
* No shortcut hacks
